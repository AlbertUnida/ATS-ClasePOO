# Documentación del Proyecto Final – Talent Flow ATS

Este directorio contiene los artefactos exigidos por la UNIDA (PF_General.docx) para el Proyecto Final. Cada unidad del plan académico se documenta en subcarpetas independientes.

## Estructura

```
docs/pf_final/
├── README.md                      # este índice
├── unidad_I/                      # Relevamiento, entrevistas, alcance
├── unidad_II/                     # Canvas, modelo de negocio, requisitos globales
├── unidad_III/                    # Casos de uso detallados, diagramas de secuencia
├── unidad_IV_V/                   # Modelo OO, DER lógico/físico, balance OO-BD
├── unidad_VI/                     # Prototipos GUI, pruebas de usabilidad
├── uml/                           # Diagramas en formato PNG/PDF/PlantUML
├── der/                           # Diagramas entidad–relación
├── planificacion/                 # Gantt, cronograma, riesgos, viabilidad
├── entrevistas/                   # Minutas y cuestionarios levantados
└── referencias_APA.md             # Bibliografía y citas
```

## Próximos pasos
1. Migrar cada artefacto existente (Word, PDF, imágenes, Figma) a las carpetas correspondientes.
2. Crear versiones Markdown o PDF para facilitar revisión en Git.
3. Mantener trazabilidad: todo cambio debe incluir fecha y autor en cada documento.
4. Incluir versiones finales en `docs/pf_final/` antes de la defensa.

> **Nota**: Los recursos originales (`PF_General.docx`, notas internas, etc.) se migrarán a este repositorio para que el tribunal pueda auditar el proceso completo.
